apt-get mini-COMO

Este documento es una bréve aproximación a las utlidades del ``apt-get'', interfaz en línea de órdenes de APT (Advanced Package Tool), herramienta para la gestión de paquetes de software del Proyecto Debian.

La última versión de este mini-COMO la puedes conseguir en http://www.ctv.es/USERS/vigu/linux/docs/apt-get-mini-COMO.tar.gz

Gracias a toda la gente de la lista de correo de Debian en español por las orientaciones sobre el uso y abuso de ``apt'', ¡sois geniales!.

1. Limitación de responsabilidades de uso de este miniCOMO.

Si te pasa algo malo en tu sistema usando los consejos que aquí se dan, lo siento pero te lo advierto ahora: esto no está exento de fallos seguro, así que leelo y úsalo bajo tu responsabilidad.

2. ¿Qué es...

2.1 ... apt?.

APT son las siglas de Advanced Package Tool. APT es un sistema de gestión de paquetes de software desarrollado por el APT Team del Proyecto Debian bajo licencia GNU/GPL.

Es el sistema de gestión de paquetes más avanzado en la actualidad y el que má flexibilidad y potencia posee para entornos de red.

2.2 ... apt-get?.

``apt-get'' es la utilidad para usar desde la línea de órdenes que hace de interfaz entre el usuario y APT.

3. Modo de uso de ``apt-get''.

3.1 Indicación de lugares desde los que se toman los paquetes.

Editar ``/etc/apt/sources.list''. En este fichero se guardan las direccio nes que usará ``apt'' para al hacer un ``apt-get'' tomar los paquetes a actualizar o instalar por primera vez.

3.2 Actualización de la lista de paquetes.

Ejecutar ``apt-get update'' para actualizar la lista de paquetes, ``apt'' descargará descripciones de los lugares indicados en ``/etc/apt/sources.list''.

3.3 Actualización de paquetes.

Según lo que se quiera hacer:

Actualizar o instalar por primera vez algún paquete:

Ejecutar ``dselect'' y elegir los paquetes que se quieran o bien ejecutar ``apt-get upgrade'' para instalar las nuevas versiones de los paquetes ya instalados. Seguir los pasos de ``dselect'': instalar, configurar, borrar y salir.

Si solo es un paquete puede ser más rápido: ``apt-get upgrade paquete''.

Actualizar toda la distribución.

Ejecutar ``apt-get dist-upgrade''.

4. Configuraciones para el ``/etc/apt/sources.list'', todas con Debian 2.2.

Lo de que ponga 2.2 y no 2.1 es para que este mini-COMO no se quede viejo rápidamente ;-)

4.1 Sencilla, con mirror de non-US.

  • deb http://ceu.fi.udc.es/debian unstable main contrib non-free
  • deb http://ceu.fi.udc.es/debian-non-US unstable/non-US main contrib non-free

4.2 Doble, mirror rápido primero y lento segundo, sin non-US.

  • deb file:/home/ftp/pub/debian potato main contrib non-free
  • deb file:/home/ftp/pub/debian non-US main contrib non-free
  • deb http://www.debian.org/ ljlane/downloads enlightenment-cvs/
  • deb ftp://ftp.debian.org/pub/debian potato main contrib non-free

4.3 Sencillo, con descarga de código fuente opcional (de Jaime E. Villate, un tipo a tener muy en cuenta).

Yo tengo el ``/etc/apt/sources.list'' así:

  • deb ftp://reinolinux.fe.up.pt/pub/debian unstable main contrib non-free
  • deb-src ftp://reinolinux.fe.up.pt/pub/debian unstable main contrib non-free

Y para bajar fuentes uso:

  • apt-get source paquete

5. Algunas direcciones útiles para añadir al ``/etc/apt/sources.list''.

5.1 GNOME para Slink.

  • deb http://www.debian.org/ vincent/ slink-update main

Instrucciones para instalación:

  • apt-get update
  • apt-get install task-gnome-apps

5.2 KDE para Slink.

  • deb http://kde.tdyc.com/debian/ potato kde kde2 contrib rkrusty
  • deb http://debian.tdyc.com/debian/ potato kde kde2 contrib rkrusty
  • deb ftp://sunsite.tut.fi/pub/Mirror/debian/ruins.tdyc.com/pub potato kde kde2 contrib rkrusty
  • deb ftp://ftp.uni-marburg.de/mirror/debian-kde.tdyc.com/ potato kde kde2 contrib rkrusty
