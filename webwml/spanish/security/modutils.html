Información sobre Seguridad

Por favor, envie las noticias de fallos relacionados con la seguridad a security@debian.org

Paquetes afectados:
modutils
Breve descripción:
request-route utiliza un fichero de bloqueo (lock file) en /tmp
Vulnerable:
Sí
Corregido en:
modutils 2.1.34-5a, pero utilice 2.1.34-8 o posterior si está disponible
Nota:
El uso de request-route no es recomendable. El paquete diald ofrece la misma funcionalidad de un modo mucho mejor. En un kernel futuro, el soporte para request-route será omitido.
