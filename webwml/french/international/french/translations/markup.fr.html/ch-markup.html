Manuel de Debiandoc-SGML - Chapitre 1
Balises SGML et caractères spéciaux

Puisqu'il est saisi avec un éditeur de texte, un document SGML est constitué de texte entremêlé de balises marquant le début et la fin d'éléments.

Les balises sont de la forme <element> pour commencer l'élément element et </element> pour le finir.

Il y a quelques raccourcis que vous pouvez utiliser: <element/contenu/ spécifie un élément element qui contient contenu - mais le contenu ne peut pas contenir une barre oblique /. </> ferme l'élément encore ouvert le plus proche.

Certains types de balises de début d'élément peuvent avoir des attributs; ceux-ci apparaissent à l'intérieur des crochets angulaires et sont séparés du nom de l'élément par un espace. Les attributs autorisés pour une balise de début d'élément particulière sont décrits en même temps que cet élément.

Si vous voulez inclure des caractères spéciaux SGML (crochets angulaires < >, et commercial &) comme texte, vous devez les désigner par leurs noms (ceci est appelé une référence d'entité dans le jargon SGML). &name; produit le caractère dont le nom est name. Quelques noms de caractère utiles sont:

lt
le symbole inférieur (crochet angulaire gauche) <;
gt
le symbole supérieur (crochet angulaire droit) >;
amp
le et commercial &;
copy
le symbole copyright ©;
La liste complète des caractères est empruntée à linuxdoc-sgml, dans le fichier rep/latin1/general.

Vous pouvez aussi utiliser &#number; pour désigner le caractère dont le numéro est number (en ISO-LATIN-1). number doit être en décimal.
