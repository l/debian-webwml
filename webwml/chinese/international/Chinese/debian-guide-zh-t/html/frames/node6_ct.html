2 開始

``A journey of a thousand miles must begin with a single step.'' - 老子
現在你已經讀過有關 Linux 和 Debian 的理念與哲學，現在是時候把它搬上你的電腦上了！我們會首先講及怎樣準備安裝 Debian，然後是怎樣分割硬碟，最後是怎樣開始安裝系統。


1 硬體支援

Debian 沒有強迫硬體要求雖要超越 Linux 核心及 GNU 工具。

這個章節包含慨括性資料和找到附加資料的指標而不是嘗試形容所有被 PC 平台支援的不同硬體設定。

這裡有兩個一流的地方去查詢更詳細的資料：Debian 系統需求 列表和 Linux 文件計劃的硬體相容性 HOWTO (Hardware Compatibility HOWTO) 。至於顯示卡的支援，你可以觀看 XFree86 計劃的網站。


1 記憶體及硬碟空間需求

你需要最少 4MB 的記憶體和 35MB 可用的硬碟空位。如果你想安裝適量的軟件，包括 X 視窗系統，和一些開發用的程式以及函式庫，你需要最少 300MB。而一個完整的安裝則需大約 800MB。要安裝所有在 Debian 裡的東西，你大概需要 2GB。事實上，安裝所有的東西是沒有意義的，因為有些軟件包提供相同的功用。

2 在你開始之前

在你開始前，請先確定你已經備份好所有系統上的檔案。安裝過程可能會去除所有在硬碟上的資料！安裝過程中使用的程式都是很可靠的，而且大部份都已經用了很多年，但一個錯誤的決擇所換來的代價可不少呢。就算在備份過後，請小心地想好你要回答的問題及行動。兩分鐘的思考將會節省數以小時的無謂工作。

Debian 允許你同時安裝及擁有 Debian GNU/Linux 和其它作業系統在相同的電腦上。如果你計劃使用這個選項，請確定在你手上已擁有其它已安裝的作業系統的光碟或磁碟。如果你重新分配啟動硬碟機的分割區，你可能需要重新安裝你已有的作業系統啟動載入器或者整套作業系統。

1 你所需要的資料

如果你的電腦是二十四小時連接上網絡（以太網或相似的本地網絡連接──不是 PPP 撥號連接），你應該詢問你的網絡的 管理員以下的資料：

  • 你的機器名稱（或許你能自行決定）
  • 你的網域名稱
  • 你的電腦使用的 IP 位址
  • 網絡的 IP 位址
  • 網絡使用的子網絡遮罩 (netmask)
  • 網絡使用的廣播地址 (broadcast address)
  • 如果你的網絡有閘道，你便需要填上預設閘道的 IP 地址
  • 網絡上使用的 DNS 伺服器
  • 你是否使用以太網去連接網絡？
  • 你是否使用 PCMCIA 以太網配接卡？那麼哪一類控制器是你使用的？
如果你只是使用電話線 PPP 撥號連接或相類似的連接方法，你便不需要擔心安裝過程中的網絡設定。在第 頁的第 11.1 節會有關於在 Debian 下設定 PPP 撥號連接的資訊。


3 建立硬碟分割區

在你安裝 Debian 之前，最好請先計劃一下怎樣安排你的硬碟內的資料。這一部份的過程涉及在你的硬碟上建立分割區。


1 背景

分割你的硬碟就是簡單地指在你的硬碟上分柝開幾個部份的動作。每個部份之間是獨立及沒有關係的。這動作大概就像在屋中築起牆子；之後，在房間內加入傢具便不影響其它房間一樣。

如果你在你的系統內已經擁有一個作業系統（Windows 95, Windows NT, DOS 等）而你亦想把 Debian GNU/Linux 安裝在同一隻硬碟上，你便需要再重新分配硬碟分割區。概括來說，在一個已建立檔案系統的分割區上作出改變是會破壞所有在該檔案系統內資料。所以你應該在你重新分配硬碟分割區之前做好備份。如用屋的比喻，就是你大概會在你移動牆子之前先搬出所有傢具去避免移動牆子時做成破壞傢具的危險。幸運地這裡有一個另外的方法給某些使用者使用；詳細資料請看在第 頁的第 2.3.6 節。

在最低限度，GNU/Linux 自已雖要一個分割區。你可以在一個分割區內裝有作業系統，應用程式和你的私人檔案。但是，大多數人都會選擇給 GNU/Linux 多過最少數目的分割區去使用。主要有兩個原因去解釋為什麼要將檔案系統去分成數個分割區。第一個原因是為了安全。如果有什麼損壞檔案系統的事情發生，只會影響一個分割區。所以，你只需要去替換（由你小心保存好的備份）那一部份的系統。最少限度，你應該考慮建立被稱為「根分割區」。這個分割區包含系統上最緊要的元件。就算其它的分割區損壞，你仍然可以啟動進入 GNU/Linux 去修復系統。這樣可以使你免於重新安裝系統的麻煩。

第二個原因對於商業應用較為重要，但這項也要取決於你怎樣使用系統。假設有什麼失控情況和開始蝕掉硬碟空間。如是程序引致擁有根權限的問題（系統保留一定數量百份比的室間），你可能會發覺沒有足夠的硬碟空間。這並不是一個好的情況，因為作業系統雖要使用硬碟（除了交換空間）來工作。這個可能不是一個在本地區所發生的問題。例如，多餘的電子郵件 (``spam'') 可以輕易地填滿一個分割區。透過使用多個分割區，你可以保護很多系統上的問題。再次使用電子郵件的例子，把 /var/spool/mail 放在獨立的分割區，就算多餘的電子郵件填滿了整個分割區，大部分系統都會繼續工作。

另外一個原因只會發生在你擁有一個大型的 IDE 硬碟機但又沒有使用LBA 定址或者驅動程式。在這個情況，你需要把根分割區放進首 1,024 磁柱內，通常大概在 524 megabytes。關於詳細資料請看在第 頁的第 2.3.3 節。

很多人都應為交換分割區是必須的，雖然這不是硬性規定。「交換分割區」是一堆作業系統使用的空間，作用是允許系統使用硬碟機儲存為有別於普通記憶體的「虛擬記憶體」。把交換區放在分開的分割區容許 Linux 在使用時更有效率。要強迫 Linux 使用單一檔案為交換區是可行的，但不建議這樣做。

唯一使用多點分割區的真實缺點是通常很難預計實際需要。若果你把分割區設定得太少，你可能雖要重新安裝系統或者你需要時常搬動資料去在細小的分割區上獲得空間。反過來說，如果你把分割區設定得太大，你可能會浪費一些在其它地方用得著的空間。


2 計劃系統的用途

你選擇的安裝種類將會影響需要的硬碟空間和你的分割區方案。

為了更加方便，Debian 提供數個預設的「範本」(profiles)。部份將會在本章節中列出。範本是一些事先選擇好的軟件包，作用是為了在系統上提供想要的功能。安裝的過程也變得容易些，因為那些符合靠本要求的軟件包已經自動選擇好。每個範本會列出最後當安裝過程完成後的空間。就算你不打算使用那些範本，討論對於計劃非常重要，因為你會從討論中獲得設定分割區大小，或需要的分割區常識。以下是一些現有的範本和它們的大小：

Server_std.
這個是小型伺服器，適用於剝落了的伺服器，沒有花巧，是給 shell 使用者的。基本上有 FTP 伺服器、萬維網伺服器、DNS、NIS 和 POP。這個設定使用 50MB。當然這只是軟件的大小；所有你伺服的資料會增加使用的空間。
Dialup.
這個範本是一個好的標準桌面環境，包括 X 視窗系統，圖像程式、聲音、編輯器等。這些軟件包需要大概 500MB。
Work_std.
這個範本適合一些盡量細小，並不需要 X 視窗系統或 X 應用程式的機器。這個範本亦適合於小型或流動電腦。需要大約 140MB。其實是有可能利用少於 100MB 的空間來為小型電腦安裝一個包括 X 的簡單設定的。
Devel_comp.
這個是桌面設定範本包括有名的開發軟件包，像 Perl, C, 和 C++。需要大約 475MB。假設你需要加入 X 和額外的軟件包作其它用途，你需要大概 800MB 的空間。
請記著這些空間並不包括其它在系統上普遍找到的物料，包括使用者檔案、郵件和資料等。請最好大方地考慮空間分配給檔案及資料。明顯地，Debian 的 /var 目錄內包含很多的狀態資訊。已安裝的軟件包管理檔案會消耗大約 20MB 的硬碟空間。概括來說，你應該為 /var 目錄預留最小 50MB 的硬碟空間，因為系統的記錄檔案也會存放在那裡。


3 個人電腦磁碟的限制

個人電腦的基本輸入輸出模組會為硬碟分割時增加額外的限制條件。這個就是可擁有有多少個主要分割區及邏輯分割區。而且，輸入輸出模組在硬碟上的哪個部份去讀取啟動資訊也是有限制的。更多的資料可以在 Linux 分割區 mini-HOWTO 找到。這個章節會幫你概括地縱覽如何設計大多數的情況。

「主分割區」(primary partitions) 是個人電腦硬碟上的原本分割方案，但是只可以擁有四個。要避過這種限制，就需要有「延伸」(extended) 或「邏輯」(logical) 分割區。透過將一個主要分割區設定成延伸分割區，你可以分拆那些空間成為邏輯分割區。可建立的邏輯分割區的數量較可建立的主分割區的限制少得多，但你只可以在一個硬碟機上擁有一個伸延分割區。

Linux 限制了在一個硬碟上的分割區數目。SCSI 硬碟機可以有 15 個分割區，3 個可用的主分割區，12 個邏輯分割區。IDE 硬碟機可以有 63 個分割區，3 個可用的主分割區和 60 個邏輯。

最後一個你需要知道關於輸入輸出模組的因素就是你的啟動分割區，這個分割區包含你的核心影像──需要放在硬碟上首 1024 個磁柱內。因為根分割區通想都是你的啟動分割區，你需要確定你的根分割區在首 1024 磁柱內。

如果你有一個大的硬碟，你便可能需要使用磁柱轉換 (cylinder translation) 的技術，你可以在輸入輸出模組設做設定，例如 LBA 轉換模式。（在大容量硬碟 mini-HOWTO (Large Disk mini-HOWTO) 內可找到更多有關大容量硬碟的資訊。）如果你選擇用磁柱轉換的方法，你的起動分割區就必須要在已轉換的 1024 個磁柱之內。


4 Linux 的裝置名稱

Linux 硬碟機和分割區的名稱都是和其它作業系統不同的。你應該要知道當你建立加載分割區那些 Linux 用的名字。基本的配置可以在第 頁的表 2.1 中找到。

Table 2.1: Linux 裝置名稱
裝置 Linux 所用的名稱
第一台磁碟機 /dev/fd0
第二台磁碟機 /dev/fd1
在 /dev/hda 的第一個分割區（在其它作業系統裡通常是 C:） /dev/hda1
在 /dev/hdc 的第五個分割區 /dev/hdc5
在 /dev/sdb 的第二個分割區 /dev/sdb2
整個在 Primary-Master 的 IDE 硬碟機或光碟機 /dev/hda
整個在 Primary-Slave 的 IDE 硬碟機或光碟機 /dev/hdb
整個在 Secondary-Master 的 IDE 硬碟機或光碟機 /dev/hdc
整個在 Secondary-Slave 的 IDE 硬碟機或光碟機 /dev/hdd
第一台 SCSI 硬碟機 /dev/sda
第二台和其它餘下的 SCSI 硬碟機 /dev/sdb，如此類推
第一個串連埠（在其它作業系統上叫 COM1） /dev/ttyS0
第二個，第三個，及以後的串連埠 /dev/ttyS1, /dev/ttyS2, 等等
SCSI 的磁帶機（自動回帶） /dev/st0, /dev/st1, 等等
SCSI 的磁帶系統（沒有自動回帶） /dev/nst0, /dev/nst1, 等等
SCSI 光碟機 /dev/scd0, /dev/scd1, 等等


在每台硬碟機上的分割區都會附在硬碟機名稱加一個數字去表示。例如 hda1 和 hda2 等名稱就代表第一台 IDE 硬碟上的第一及第二個分割區。Linux 會用硬碟機名稱以及分割區數字由 1 至 4 來表示主分割區。例如第一個在第一台 IDE 硬碟機的主分割區是 /dev/hda1。邏輯分割區會從數目字 5 開始，所以第一個在相同硬碟機上的邏輯分割區便是 /dev/hda5。請記住那個伸延分割區──就是盛載著邏輯分割區的主分割區是不可以便用的。無論是 SCSI 抑或 IDE 硬碟機也是如此。

現在假設你有兩台 SCSI 硬碟機，一台在 SCSI 位址 2 和另外一台在 SCSI 位址 4。第一台及第二台硬碟機（在位址 2 及 4）就會分別被命名為sda 和 sdb。如果 sda 硬碟機內有三個分割區，它們會被稱為 sda1，sda2 及 sda3。同樣道理也適用在 sdb 硬碟以及它的分割區。如果你擁有兩片 SCSI 控制卡 (SCSI host bus adapters/controllers)，硬碟的次序就會變得混淆了。在這個情況下，最好的解決方法就是觀看啟動時的訊息（假設你清楚硬碟機的型號）。


5 建議的分割區規劃

就如上述，你應該有一個細小及分開的根分割區以及當你有足夠空間，有一個大的 /usr 分割區。就大多數的使用者來說，以上說的分割區方案已經足夠。以上的方案特別適合那些只有單一，而且細小的硬碟，因為建立更多的分割區會浪費空間。

在某些情況，如果你計劃安裝大量非 Debian 分發版本的程式，你可能需要分割 /usr/local 分割區。如果你的機器會成為一台郵件伺服器，你可能需要建立一個/var/spool/mail 的獨立分割區。把 /tmp 設定成一個 20 至 32MB 的獨立分割區是一個好主意。如果你打算設定成一台多使用者賬戶的伺服器，你需要一個大的 /home 分割區去儲存使用者的目錄。總括來說，每台電腦的分割區分佈情況都跟它的用途而不同。

對於非常複雜的系統，你應該看多重硬碟 HOWTO (Multi Disk HOWTO) 。裡面包含更深入的資訊，非常適合要設定伺服器的人仕。

交換分割區的大小也需要考慮。對於交換分割區的大小，有很多觀點。一個成功的經驗做法，就是你有多少系統記憶體就用多少交換分割區。雖然交換分割區很少情況會大過 64MB，但是在多數的請況下也不要少於 16MB。當然這些都有例外，如果你嘗試用一部有 256MB 記憶體的電腦去解決 10,000 條聯立方程式，你可能需要 1GB（或更多）的交換空間。

舉個例子，試想像有一台擁有 32MB 記憶體的電腦和 1.7GB IDE 硬碟在 /dev/hda。在 500MB 的 /dev/hda1 分割區內有另一個作業系統。 一個 32MB 的交換分割區在 /dev/hda3， 另多一個大約 1.2GB 的 Linux 分割區，在 /dev/hda2。

6 在安裝前分割硬碟

你有兩個時間可以去分割硬碟：在安裝 Debian 前或安裝 Debian 中。如果你的電腦只裝上 Debian，你應該跟據第 頁第 3.5 節所說的一樣，在安裝 Debian 中分割硬碟。如果你的電腦有多過一個的作業系統，你應該樣其它作業系統建立自已的分割區。

下面幾節包括怎樣在安裝 Debian 之前，從已有的作業系統分割硬碟的資料。要注意的是你需要配對其它作業系統與 Linux 作業系統的分割區名稱，請看在第 頁的表 2.1。

1 在 DOS 或 Windows 下分割硬碟

如果你處理現有的 FAT 或 NTFS 分割區，我建議你使用以下的方案或者 Windows 及 DOS 的工具。否則，便不需要在 DOS 或 Windows；在 Linux 上的硬碟分割工具會做得更好。


2 無損性重組硬碟分割區

其中一個最常見的安裝方法就是安裝在一個已經有 DOS (包括 Windows 3.1)，Win32（例如 Windows 95, 98, NT），又或者 OS/2，而且在不損害以前的系統下，把 Debian 放在相同的硬碟上。如在第 2.3.1 頁的第 2.3.1 節中所說，除非做好預防工作，減少現有的分割區的大小會損害在分割區上的資料。在這裡述的方法並不保證可以保護你的資料，但實際效果非常之好。為以防萬一，請務必做一個備份。

在你想進行下一步前，你應該要決定一下要怎樣分配你的硬碟。我在這裡使用的方法只會分成兩個分割區，一個包含原本的作業系統，另外的會被 Debian 使用。在安裝的時候，你會有機會選擇怎樣使用餘留給 Debian 的那部份，例如是用作 swap 還是檔案系統。

大意就是在改變分割區資料前搬動所有在分割區上的資料去到前端，所以沒有東西會被遺失，重要的是你應該盡量避免搬動資料及再分割磁碟去減少檔案被寫在接近分割區的末段，這樣會減少你在分割區上的總共空間。

你需要的第一樣東西是一個叫 FIPS 的程式。你可以在 Debian CD-ROM 內 tools 的目錄中找到。這片磁碟一定要是可啟動的。在 DOS 環境下，一片可啟動的磁碟可以用 sys a: 來建立在已格式化的磁碟上或者用 format a: /s 在未經格式化的磁碟上。把壓縮檔解壓，並把 RESTORB.EXE、FIPS.EXE 和ERRORS.TXT 複製到可啟動的磁碟。FIPS 的程式包含很好的文件，這些可能是你想閱讀的。尤其是你使用磁碟壓縮的驅動程式或者磁碟管理程式時，更必須明確地閱讀那些文件。在您繼續前請建立好磁碟和閱讀說明文件。

接著要做的事情就是搬動所有資料到分割的開端。DEFRAG，包含在DOS 6.0 或以後版本，可以輕鬆地完成這項工作。請閱覽 FIPS 的文件說明檔內其他可使用的軟件列表。要注意的是如果你使用視窗 95 或更高的版本，你應該在視窗內執行 DEFRAG，因為 DOS 不明白 VFAT。VFAT 是用來在視窗 95 或以上版本內支援長檔名稱的。

執行過重組程式（在較大的磁碟上使用多點的時間）後，使用你建立的 FIPS 磁碟重新啟動。只須鍵入 a:\fips，再跟隨指示即可。

如果 FIPS 不能為你工作，你可以選擇其它分割磁碟管理程式。

7 Debian 的安裝過程

在你開始安裝 Debian，你會實行以下幾個不同的步驟：

  1. 啟動系統 (Boot the installation system)
  2. 啟始系統的設定 (Initial system configuration)
  3. 安裝基本系統 (Install the base system)
  4. 啟動新安裝的基本系統 (Boot the newly installed base system)
  5. 安裝餘下的系統 (Install the rest of the system)
啟動 Debian 安裝系統，也就是第一個步驟，大概就是用光碟裡的救援磁碟。

當你已經啟動到 Linux，dbootstrap 程式會被執行以及帶領你進入第二個步驟，就是啟始 系統的設定。這個步驟將會在第 頁的第 3 節再詳細討論。

``Debian base system'' 是一些重要且需要用來啟動 Debian 的基本軟件包。Dbootstap 會安裝那些軟件包，就像在第 頁的第 3.12 節內描述的一樣。當你已設定及安裝好基本系統，你的機器便可以「獨立運作」了。

最後一步就是安裝餘下的 Debian 系統。這些包括將會被你使用的應用程式和文件說明檔。例如：X 視窗系統、編緝器、指令介面和開發環境。餘下的 Debian 系統可以由光碟機安裝。在這時，你將會使用 dselect 一類的標準 Debian 軟件包管理工具。 這個步驟將會在第 頁第 3.20 節內詳述。


4 選擇你的安裝媒體

首先，選擇安裝系統的啟動媒體。下一步，選擇在安裝基本系統時使用的方法。

要去啟動安裝系統，你有以下的選擇：由可啟動的光碟機，磁碟或者非 Linux 的啟動載入器。

光碟機啟動是一種最容易的安裝方法。並不是所有機器能夠直接經由光碟機啟動，所以你可能仍需要使用磁碟。大部分平台支援由磁碟啟動。磁碟啟動會在第 頁第 2.4.2 節中詳細描述。


1 經由光碟機安裝

如果你的系統支援經由光碟機啟動，你並不需要任何磁機。把光碟方入光碟機，關掉你的電腦，接著開啟。你應該會看見一個在畫面下端有 boot 的字元提示器的歡迎畫面。 現在，你可以跳到第 2.5 節。

如果你的電腦「看」不見 Debian 光碟，最簡單的方法就是做兩隻用來啟動的磁碟（在第 2.4.2 節詳述）接著使用它們在啟動 Debian。不用擔心，當 Debian 完成載入那兩片磁碟，它便能順利地找到你的光碟。


2 由軟磁碟啟動

由磁碟啟動並不是一件難事，事實上，你的光碟內包含了所有需要去建立啟動磁碟的資訊。經那些資訊，你將會需要兩片磁碟。第一片寫上「Debian 2.1 安裝/救援磁碟」和第二片「Debian 2.1 模組/驅動程式磁碟」。


1 由磁碟影像建立軟磁碟

磁碟影像就是一些包含了一片磁碟內完整內容的檔案以及用原始 (raw) 方式儲存。磁碟影像，例如 resc1440.bin，不可以簡單地後裝到磁碟機中。一個特別程式，會被使用來把影像檔案用原始方式寫入磁碟。

首先，你需要一個 DOS 字元提示器。在視窗 95 或以上版本中， 你可以雙擊 MS-DOS 圖示或者去開始程式集MS-DOS 模式。接著插入你的 Debian GNU/Linux 光碟機中。首先，你要轉換到光碟機中。在大多數的情況下，這是 D:。

C:\WINDOWS>D:
現在，轉換在那個包含磁碟影像的目錄中。

D:\>CD 

\DISTS\SLINK\MAIN\DISKS-I386\2.1.8-1999-02-22

如果你得到錯誤，雙重檢查你鍵入的資訊。如果錯誤仍然存在，用人手輸入 CD \DISTS\SLINK\MAIN\DISKS-I386，接著執行 DIR，接下來再 CD 進入提示的目錄。請注意以上的指示，以及下面某些其他範例，可能會以單行出現，就算在這裡被繼成兩份。

現在，你準備去建立兩片中的第一片磁碟。啟始程式去寫，用 rawrite2:

D:\DISTS\SLINK\MAIN\DISKS-I386\

2.1.8-1999-02-22>rawrite2

RaWrite 2.0 - Write disk file to 

raw floppy diskette

Rawrite2 開始 以及顯示它的歡迎訊息。接著，它用問及檔案名稱和磁碟機編號。你告訴它去把resc1440.bin 寫進 a:

Enter disk image source file name: resc1440.bin

Enter target diskette drive: a:

Rawrite2 現在會叫你把磁碟插入磁碟機中。插入並按 Enter。

Plese insert a formatted diskette into 

drive A: and press -ENTER- :

在這裡，rawrite2 將會建立兩片中第一片磁碟。現在，你需要在第二片磁碟，重複整個步驟。

D:\DISTS\SLINK\MAIN\DISKS-I386\

2.1.8-1999-02-22>rawrite2

RaWrite 2.0 - Write disk file to

raw floppy diskette

Enter disk image source file name: drv1440.bin

Enter target diskette drive: a:

Please insert a formatted diskette into

drive A: and press -ENTER- :

現在，建立了兩隻磁碟。你可以使用第一隻去啟動。

2 啟動 Debian

你現在可以準備啟動進入 Debian！關掉你現在運行中的作業系統，關上電腦，以及放入安裝/救援磁碟去磁碟機。現在啟動你的電腦。你應該一個有 boot 字元提示器在下方的歡迎畫面。


5 啟動安裝系統

你現在應該見到 boot 提示標。在這裡你只需按 Enter。

當你按了 Enter，你應該看到 Loading... 訊息，接著是 uncompress Linux...，再來是一連串關於你的系統內硬件的信息畫面。概括來說，你可以略過這些訊息。Linux 會搜尋一連串的硬件裝置和告訴你那些被找到以及那些不被找到。你現在不需要擔心那些訊息。你只需要等待一個彩色的選擇畫面。若你遇到麻煩，請看第 頁的附錄 B.2。
