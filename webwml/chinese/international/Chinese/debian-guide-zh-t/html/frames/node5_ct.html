1 導引

我們很高興有這個機會向您介紹 Debian！在我們一直往 GNU/Linux 的路上前進的時候，我們會首先簡介何謂 Debian──它做甚麼，和它在自由軟件(Free Software) 這個浩瀚世界中的定位。之後，我們會簡述自由軟件這個現象，以及它對於 Debian 和您們的意義。最後，我們會講講有關這本書的一些資料來結束這一章。


1 Debian 是什麼？

Debian 是為你的電腦所設計的一個自由 (free) 作業系統(operating system, OS)。作業系統是使電腦運行的基本程式和工具的集合，其中最主要的部分稱為核心 (kernel)。核心是電腦中最基要的程式，負責一切基本「持家」的調度工作，並讓你執行其它程式。Debian 使用 Linux 核心，它是一個完全自由的軟件，由 Linus Torvalds 開始編寫，並由世界各地數以千計的程式編寫員支援。作業系統裡的大部份基本工具都是來自GNU 計劃 ，而這些工具也是自由軟件。

一個作業系統的其中一面是應用軟件 (application software)：一些幫助你完成工作的程式，由編輯文件至進行商業交易，由遊戲玩樂至編寫更多的軟件。Debian 擁有超過 1,500 個軟件包（已編譯的軟件，並且把它包裝了起來以便安裝）──這些全部都是自由軟件。

Debian 系統有點像一個金字塔。在它的基底是 Linux。之上是所有的基本工具，主要是來自 GNU。在其上是你在機器上執行的所有應用軟件，有不少也是來自 GNU 的。Debian 的開發人員好比建築師和協調者──他們把所有的東西適當地結合起來，小心地把系統組織好而製成一個穩定的作業系統：Debian GNU/Linux。

GNU/Linux 的設計哲學是把它的功能用一些細小但多用途的元件來分佈。這樣做你就可以將這些細小的元件（程式）用新的方式組合而達到新的功能。Debian 就好像一個 erector set：你可以用它來建造任何東西。

當你使用一個作業系統時，你定想付出最少的精神時間來完成你的工作。Debian 附有很多工具來幫你，不過你一定要知道這些工具是做甚麼的。虛耗一個小時嘗試使用某些東西但最後放棄了，功虧一簣，是非常沒有工作效率的。本書會教導你那些組成 Debian 的基本工具：在甚麼情況下用甚麼工具，和怎樣把這些工具組合起來一齊使用。

1 誰製作 Debian？

Debian 是一個完全由志願工作者製作，並利用互聯網 (Internet) 來開發的計劃。現正有數以百計的志願工作者在發展它，他們很多都是負責一小部份他們非常熟悉的軟件的軟件包。

這些志願工作者需要嚴守一些準則來規定怎樣組合一個軟件包，而這些準則都是大家一起在 Internet 的郵遞列表 (mailing list) 上經過多番討論才建立出來的。

2 多用者、多工 (multiuser, multitasking) 的作業系統

我們在第 1.1 節提過，Debian GNU/Linux 的設計是源自 Unix 作業系統的。和一般桌面用的作業系統如 DOS、Windows 和 MacOS 不同，GNU/Linux通常是使用在大型的伺服器和多用者的系統。

因為這樣，Debian 擁有其它作業系統沒有的特點。它容許在同一時間有很多的人使用同一部機器，只要每個用者都有他的一個終端 (terminal)。為了使不只一個用者能在同一時間工作，Debian 需要令很多程式可以一同執行，這個特點叫做多工作業。

GNU/Linux 系統的強大之處（和她的複雜性）大部份都是源自這兩個特點。例如，系統必須用某些方法來防止使用者不小心地刪除了別人的檔案。系統亦需要協調正在運行中的不同程式不會在同一時間使用了一樣的系統資源，例如硬碟機。

如果你能緊記 Debian 設計的原意，在很多方面你就不會覺得奇怪的了，而你亦將會學習怎樣去掌握並盡量利用這些特點的能力。

3 甚麼是自由軟件 (Free Software)？

當 Debian 的開發人員和使用者說 ``Free Software''，他們其實是指 freedom（自由）而非價錢。在 Debian 的范疇裡，自由的意思是：你可以自由地更改及分發它，並且，你永遠也可以為了這些原因而得到它的原代碼 (source code)。Debian 自由軟件指導標準 (Free Software Guidelines) 詳細說明「自由」的意思。GNU 計劃的始創組織，自由軟件基金會 (Free Software Foundation) ，是另一個擁有很多好資料的地方。你可以在 Debian 的網頁 裡找到更多有關自由軟件的討論。在這個范疇裡其中一個最出名的著作是Richard M. Stallman 的文章 Why Software Should Be Free ，閱讀這篇文章有助於明白為何我們支持自由軟件。最近，有些人開始稱呼自由軟件為「開放源碼軟件」(Open Source Software)；這兩個詞彙是互相通用的。

你可能會奇怪為何有人會利用他們那麼多的時間去編寫軟件，把它們包裝好，但最終還是把它們派發出去。這個問題的答案實在是因人以異。

很多人都相信資訊的交流及大家能互相合作的自由的重要，而他們覺得自由軟件提倡這個理念。由 1950 年代起，就已經有這樣的一個傳統來堅守這個價值觀，有時這個傳統被稱為 Hacker Ethic。Debian GNU/Linux 計劃就是基於自由軟件的自由、分享、和互相合作的理念而開始的。

亦有其他人希望學多一點關於電腦的東西，愈來愈多的人都在找尋方法去逃避專屬軟件 (proprietary software) 的昂貴價格。很多人因為欣賞他們得到的自由軟件的優點而希望能夠有所回饋，因此一個社群 (community) 就漸漸形成起來。

在學術界中有很多人製作自由軟件來協助把他們的研究成果更能廣泛地使用。商界幫助維護自由軟件，因為他們可以一起參與開發，這樣就能更快把新的功能寫到軟件中。商界亦很認同更高的穩定性，而他們又能夠選擇不同的商號負責支援的工作。

亦有人認為自由軟件是社會的一個好現象，它開放了資訊的交流，防止資訊的過份集中。當然，我們很多都覺得它是充滿樂趣的。

Debian 非常著重於自由軟件，所以我們覺得有必要用文件或其他的格式把它形式化。我們的社會契約 (Social Contract) 約定 Debian 將會是 100% 自由軟件。當你安裝一個來自 Debian 主發佈 (main distribution) 的軟件包，你可以肯定它是符合我們的自由軟件指導標準。

雖然 Debian 相信於自由軟件，但亦有人希望在他的電腦上使用專屬軟件，Debian 在任何情況下也盡量給予支持。雖然專屬軟件不屬於主發佈，它們有可能存在於 FTP 站台的 non-free 目錄中，亦有愈來愈多的軟件包的工作只是用來安裝我們不能分發的專屬軟件。

清楚分辨商業軟件 (commercial software) 和專屬軟件 (proprietary software) 是十分重要的。專屬軟件是非自由 (non-free) 軟件，商業軟件是要用金錢來購買的東西。Debian 允許商業軟件存在於主發佈，但不允許專屬軟件。請緊記，自由軟件一詞是不涉及價錢的，售賣自由軟件是絕對可以的。有關這兩個詞更加清楚的分別，請參閱 http://www.opensource.org/ 或 http://www.fsf.org/philosophy/categories.html。

4 關於本書

這本書的目標讀者是 Debian GNU/Linux 的新手。我們沒有假設你已有 GNU/Linux 或 Unix 類系統的知識，但假設你對電腦及硬件已有一些很基本的認識，例如你應知道電腦的基本組成部份，以及電腦的用途。

一般來說，這個課程嘗試幫助你去明白 Debian 系統的內部運作，希望你有能力去解決新的疑難和盡量利用你的電腦，因此我們把很多的理論和有趣的東西放到手冊的「怎樣」的層面上去。

1 怎樣閱讀這本書

對任何的電腦程式，最好的學習方法差不多都是使用它了。很多人會覺得只是看書而不用那個程式不太有用。學習 Unix 和 GNU/Linux 的最好方法就是使用它們了。只要你可以的話，就使用 GNU/Linux，儘管去試驗它吧！

對比於其他的作業系統，Debian 不是一下子就能學懂怎樣用。你或許最終也要至少看畢數章。GNU/Linux 的能力及複雜程度使它在起初的時候不太平易近人，但長遠來說是有得著的。

建議大家學習的方式是看一點，然後試一點。繼續試下去，直至你已習慣了那些概念，然後就可以隨意地跳來跳去的閱讀的了。你會發現有很多的話題也有講到，你可能會對某些有興趣。過了一會，即使你不太清楚一些指令的功用，你也會有信心去開始使用它們。這個絕對是好事。 如果你不小心打錯了一個指令，或是不知道怎樣脫離一個程式，按 CTRL-c（同時按 Ctrl 鍵和小階 c）。這樣通常能關閉程式。

2 凡例

開始之前，弄清楚本書使用字體的凡例是非常重要的。

當你要同時按數個鍵的時候，我們會用像 CTRL-a 的寫法。意思是「按 Ctrl 鍵及按小階字母 a」。有些鍵盤有Alt 和 Meta 鍵；通常一般的家庭電腦都只有Alt 鍵，但 Alt 鍵的作用就好像 Meta 鍵，所以可以試試 Alt 來代替。

像 Alt 及 Meta 鍵都被稱為更改 (modifier) 鍵，因為它們改變一些標準鍵，如字母 A 的意義。有時你需要按多過一個的更改鍵，例如 Meta-Ctrl-a 就是同時按 Meta，Ctrl 和小階 a。

一些鍵有特別的表示方式──例如 Ret (Return/Enter)、Del (Delete, 或有時是 Backspace)、Esc (Escape)。這些鍵的意思都應該不需多作解釋了。

如果不用橫劃 (hyphen) 而用空格，其意思是逐個鍵依次序按。例如 CTRL-a x RET 的意思為同時按 Ctrl 和小階 a，然後是字母 x，再跟著是按 Return。（在某些鍵盤上，這個鍵是用 Enter 標誌著，只是不同名字罷了。）

在某些章節，粗體字代表使用者鍵入的字，斜體表示那段的解釋，而其他的都是鍵入指令後的輸出。至於較短的指令，有時會用等闊度字體(monospace font) 放到其他的文字中間。
