8 使用文本文件

文本文件在 GNU/Linux 系統上應用得十分廣泛。它們包括從文檔到配置文件的所有信息。幸好，使用它們不是太難。


1 查看文本文件

一個文本文件其實就是一個包含對人可讀的文本的標準文件。除此之外，沒有其它特殊的地方。另一種類型的文件──二進制文件，則需要計算機對它進行解釋。

如果你安裝了 less 文件傳呼命令的話，你可以使用它來查看兩種中任一類型的文件，（如果你沒有安裝它，最好裝上，它非常有用）。輸入 less /etc/profile 來查看一個示例的文本文件。注意，即使文件的含義是晦澀的，你仍然可以讀出其中的字符。輸入 less /bin/ls 來查看一個二進制文件。正如你所見的，ls 程序對人而言並不可讀。

有時，你會發現以 .gz 結尾的文件。這些文件可能能夠使用 zless 來查看；你可以如下一樣運行命令：

zless /usr/doc/ae/changelog.Debian.gz

zless 對於查看文檔來說十分有用，它通常是以 .gz 格式發佈的。

這兩種文件之間的差別完全就只是其所包含內容的差別，而不同於一些其它的操作系統（如 DOS 和 MacOS），它們對不同文件的處理方式都不同。

文本文件可以包括 shell 腳本、文檔、版權通知，或者任何其它對人可讀的文本。

順便提及一下源代碼與二進制可執行文件之間的差別。/bin/ls 是一個二進制可執行文件，你可以從 Debian 下載它，但是你也可以下載一個文本文件並用它來告知計算機怎樣創建 /bin/ls。該文本文件即是源代碼。把 /bin/ls 與 /etc/profile 作比較，就會明暸如果有人想理解和修改一個軟件，源代碼有多麼重要。自由軟件提供給了你或者你的咨詢者所有重要的源代碼。


2 文本編輯器

文本編輯器是用來創建和修改文本文件內容的程序。大多數操作系統都有一個文本編輯器。DOS 有 edit，Windows 有 Notepad，而 MacOS 有 SimpleText。

Debian 提供了大量的文本編輯器。vi 和 Emacs 是兩種典型的，它們都是功能最強大而且使用最廣泛的編輯器。vi 和 Emacs 都非常複雜，需要一些操練，但是它們可以十分高效地編輯文本。Emacs 可以同時在終端和 X Window 系統下運行； vi 一般只運行在終端下，但是 vim 的變種帶有 -g 選項，這可以允許其運行在 X 下。

更簡單的編輯器包括 nedit、ae、jed 和 xcoral。nedit 和 xcoral 提供了易用的 X Window 系統圖形接口。這兒還有一些 vi 的變種。另外，你也可以找到一個 GNU Emacs 的變種 XEmacs。

本書不包括任何特殊的編輯器的詳細使用方法，可是我們將會簡要地介紹一下 ae，因為它很小，運行速度又快，而且甚至可以在 Debian 急救磁盤上找到，所以為了應付緊急情況，它是值得一學的。當你需要執行更為繁重的編輯任務時，則可以試試 vim 或者 GNU Emacs。Emacs 提供了一種出色的交互式的指南；要閱讀它，可使用 emacs 命令引導 Emacs，然後按下 F1 t 鍵。Emacs 對於關注於多用途或者編程編輯器的新用戶來說，絕對是一種極好的選擇。


3 使用 ae

你可以通過給定用戶編輯的文件名來啟動 ae，如下：

$ ae filename.txt

這將導出一個編輯器屏幕。該屏幕的頂端提供了一些快捷幫助；底端則顯示出你正在編輯的文件。在這個編輯器中移動是十分簡單的；只需使用箭頭按鍵即可。你可以通過按下 C-x C-s 來保存文件，然後你可以通過按下 C-x C-c 來退出編輯器。一旦你習慣了該編輯器，你可以按下 C-x C-h 以關閉幫助。足夠了！知道這些就可以讓你進行基本的編輯了。為了進行編程或者更細致的編輯工作，你會考慮研究一下我們在前面提到過的其它編輯器。
