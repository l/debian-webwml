10 文件系統 (Filesystems)

Debian 系統使用一個文件系統來儲存和管理你的數據。在這一章裡我們會介紹這個文件系統，讓你了解怎麼加載和移走文件系統，怎麼備份你的系統。

1 基本概念

在討論使用磁碟的技巧以前，我們最好先對一些理論上的東西進行簡單的注釋。要強調的是，你要先明白文件系統這個概念，因為它含有幾個意思，所以可能有點不好理解。

文件系統指的是整個目錄結構，我們上面提到，是從根目錄 (root) / 開始的。

一個文件系統一般是指在一個物理設備上的任何文件組織和目錄。這裡的「組織」指的是目錄層次的結構和其他所有關於文件的有用的資料：文件大小，誰有權對文件進行改動，等等的資料。你的硬碟上可能有一個文件系統，你的每個軟碟上也都有一個文件系統。

“文件系統”也可以指一個文件系統的類型。比如 MS-DOS 和 Windows 3.1 用某種方式，某種規則組織它的文件，像文件名只能有 8 個字母 3 個後綴，文件沒有關於使用限制的資料。Linux 稱這種文件系統為 msdos 文件系統。Linux 還有它自己的文件系統：ext2 文件系統（ext 文件系統第二版）。除非你需要訪問其它操作系統或有特別的需要，一般來說，你主要使用的文件系統就是 ext2 文件系統。

使用任何一個物理設備，如果你要在它上面儲存文件，則它上面必需要有一個文件系統。這裡的文件系統是上面第二種意思：一個目錄層次的結構和其他所有關於文件的有用的資料。當然任何文件系統都屬於某一個類型，所以第三個意思也有意義。如果你在一個設備裡有一個以上的文件系統，那麼這些文件系統可以是不同的類型的，例如，你可以在同一個硬碟上同時擁有一個 DOS 分區和一個 Linux 分區。

2 mount 和 /etc/fstab

這一節裡會解釋怎樣加裝一個軟盤或一個 Zip 盤，也會討論關於 /dev 目錄，介紹在多個物理設備或分區裡較好的分配目錄結構。

1 加載文件系統

在 GNU/Linux 系統上，目錄結構和物理設備間不一定是完全對應的， 這一點跟 Windows 系統不一樣，在 Windows 裡，每個驅動器（硬驅，軟驅）都有它自己的目錄結構，分別以一個字母開頭（比如 C:\）。

而在 GNU/Linux 上，每個驅動設備（一個硬盤或一個軟盤）上會有一個或多個文件系統。為了方便訪問一個文件系統，它被指定成為另外一個文件系統下的一個目錄。為了防止死循環，根文件系統（root filesystem，裡面含有根目錄 /）是不會被放在任何別的文件系統裡的，當你啟動 Debian 的時候，你可以直接訪問它。

存在於一個文件系統中而又包含另外一個文件系統的目錄，我們稱這個目錄為加載點 (mount point)。一個加載點是第一文件系統中的一個目錄，而這個目錄又“包含”了另外一個文件系統（這個系統可能在另外一個設備上，比如軟盤）。只有為把一個文件系統加載到一個加載點以後，你才能訪問那個文件系統。

比如你會把一張光盤加載到 /cdrom 這個加載點上。如果你瀏覽 /cdrom 這個目錄，你會看到這張光盤上的內容。/cdrom 這個目錄本身其實是在你的硬盤上，而就實際應用來說，這張光盤上的內容成了根文件系統的一部分，當你鍵入命令或者執行程序的時候，一個文件的具體物理位置在什麼地方對於你是沒有分別的。你可以在你的硬盤建立一個 /cdrom 目錄，放進一些文件，這些文件更如果你加載你的光碟上的文件沒有什麼區別。當你加載了一個文件系統後，你根本不用考慮物理設備的問題了。

但是，在你加載一個文件系統或者在一個沒有文件系統的設備上建立文件系統前，你需要直接使用這些設備。所有的設備都有特定的名字，這些名字是在目錄 /dev 下的。如果你現在輸入 ls /dev，你會看到一個相當長的名單，裡面有你的 Debian 系統裡所有可能的設備名稱。第 頁，表格 2.1 歸納了其中的一些設備。在你的系統裡，文件 /usr/src/linux/Documentation/devices.txt 有更完整的列表。

要加載一個文件系統，我們要讓 Linux 把它在指定設備上發現的任何文件系統與一個指定的加載點聯系起來，這裡，我們可能需要告訴 Linux 去尋找哪種文件系統。

2 例：加載一個光碟

我們在這裡演示一下加載光碟的步驟，光可以用你的 Debian 光碟（如果你是用光碟安裝的話）。因為你必須是 root 才能加載光碟，所以要盡量小心，做為 root，你擁有操縱整個系統的權力，而不僅僅是你自己的文件，水能載舟，亦能覆舟。以下命令已假定你的光驅裡有一張光碟，如果沒有，放一個進去。現在可以鍵入以下命令：

su
如果你還不是 root，你或者以 root 登錄或者使用su（super user 超級用戶）命令取得根的特權級。如果使用 su，當問你密碼的時候，輸入根密碼。

ls /cdrom
使用這個命令看一下開始以前 /cdrom 裡有什麼。如果你沒有 /cdrom 目錄，你需要用 mkdir /cdrom 建立一個。

mount
只輸入 mount 而沒有任何參數會列出目前已經加載的文件系統。

mount -t iso9660 CD-device /cdrom
在這行命令裡，你需要把你光碟設備的名稱代入 CD-device 裡，如果你不知道這個設備名稱，你的光碟有可能是 /dev/cdrom，因為 Debian 安裝過程中應該已經建立了這個符號連接。如果這個命令失敗了，試一下不同的 IDE 設備名稱：/dev/hdc 等等。你應該看到類似這個的訊息：mount: block device /dev/hdc is write-protected, mounting read-only。

參數選項 -t 指定文件系統類型，在這裡用的是 iso9660。大多數的光碟是 iso9660。 下一個參數是要加載的設備名稱，最後的參數是加載點。mount 命令有很多其他的參數可用，詳細說明參看手冊 (man mount)。

加載好光碟後，你會發現你的光驅盤打不開了，你需要先卸載光碟，才能把光碟拿走。

ls /cdrom
會証實現在 /cdrom 裡包含了你的光碟上的內容。

mount
再看一下加載的文件系統，你可以看到現在你的光驅被加載了。

umount /cdrom
這個命令 卸載你的光驅。 現在你可以安全的把你的光碟從光碟機中拿走，注意，命令 umount 中沒有 `n'，雖然英文中對應的單詞應該是 `unmount'。

exit
為安全起見，不要讓自己一直以 root 身份登錄，立刻出來。

3 /etc/fstab: 自動加載處理

在文件 /etc/fstab（縮寫：``filesystem table'' 文件系統表）裡有對你經常需要加載的文件系統的描述。這些文件系統就可以用一個比較簡短的命令進行加載，比如 mount /cdrom 就可以了。你也可以配置系統使文件系統在系統啟動時自動加載。你會想要把所以在硬盤上的文件系統在啟動時自動加載，所以 Debian 為你自動在 fstab 裡加入相應的條款。

現在輸入 more /etc/fstab，讓我們看看這個文件。它有兩條或以上條款，這些條款是你安裝時自動配置的。這個文件會是類似這樣的：

# /etc/fstab: static file system information.

#

# <file system> <mount point> <type> <options> #<dump > <pass>

/dev/hda1 / ext2 defaults 0 1

/dev/hda3 none swap sw 0 0

proc /proc proc defaults 0 0

/dev/hda5 /tmp ext2 defaults 0 2

/dev/hda6 /home ext2 defaults 0 2

/dev/hda7 /usr ext2 defaults 0 2

/dev/hdc /cdrom iso9660 ro,noauto 0 0

/dev/fd0 /floppy auto noauto,sync 0 0

第一列給出這些文件系統所在的設備，第二列是加載點的列表，如後是標明文件系統的類型。以 proc 開頭的那一行是一個特殊的文件系統。請注意交換分區（swap partition 在這裡是 /dev/hda3）是沒有加載點的，所以它的加載點那一項是 none。

最後三列需要一些注釋：

第五列是給傾卸工具 (dump utility) 用來決定什麼時候進行備份的，很多情況下，你可以在這兒使用 0。

第六列是給命令 fsck 使用的，決定在系統啟動時的文件系統檢查中，應該以什麼次序來進行。根文件系統在這項應該使用 1，不用進行檢查的文件系統（比如交換分區 swap partition）應該使用 0，其它所有的文件系統應該使用 2。注意其實交換分區 (swap) 在某種意義上來說不算一個真正的文件系統，因為它不含有文件和目錄，它只是 Linux 內核用來做次級記憶體的。但是，因為歷史原因，交換分區還是與真正的文件系統列於同一個文件裡。

第四列含有一個或以上的加載時的選項。你可以查看 mount 的手冊來了解這些選項，參見第 頁，第 5.1 節。


4 抽取式磁碟（軟碟、Zip 碟等等）

把一下幾行加入你的 /etc/fstab 文件裡：

/dev/sda1 /mnt/zip ext2 noauto,user 0 0 

/dev/sda4 /mnt/dos msdos noauto,user 0 0 

現在開始，你可以使用命令 mount /mnt/dos 來加載DOS 格式化的 Zip 碟，或者用命令 mount /mnt/zip 來加載 Linux 格式化的 Zip 碟。

如果你的系統裡有 SCSI 硬盤，你要把上例中的 sda 改成 sdb 或者 sdc。


3 備份工具 (Backup Tools)

在任何操作系統裡，備份都是非常重要的。Debian GNU/Linux 提供了幾個不同的你可能會用到的工具。此外，儘管其中很多工具本來的使用對象是磁帶備份 (tape backup)，你會發現現在它們可以用在其它方面，例如 tar 被用於在互聯網上發放程序。你會發現以下一些工具：

  • Taper 是一個菜單界面，容易掌握的備份程序。它可以使用不同介質進行備份，但它局限於不能處理太大的備份（4GB 或以上）。
  • dump 是專門為磁帶設計的，它的強項是它的文件恢復界面，低層文件系統備份，定時漸進式備份。它的局限是不能備份 NFS (network filesystem 網絡文件系統) 或者 其它非 ext2 的文件系統，而且它的一些預設值也不太合理。
  • GNU tar（縮寫：Tape ARchiver 磁帶歸檔程序）是一個可能是在 Linux 界被最廣泛使用的備份和歸檔工具。它是一個很好的多用途工具，可以用於最多的目標媒質上。又因為很多系統都可以讀取 tar 文件，所以它具有非常好的可攜性。它的弱點是在漸進式備份上不如 dump，也沒有一個互動的檔案回復操作界面。


1 tar

因為 tar 非常的有用，而且在備份以外還有很多其他的功用，所以我們在這裡對它進行一些注釋。更詳細的介紹當然是參看手冊 (man tar)。使用手冊的方法在第 頁，第 5.1 節中介紹。

tar 是一個歸檔程序，就是說 tar 可以把許多文件聯合包裝成一個大的文件或者把它們寫入一個備份設備，例如一個磁帶驅動器。當你生成了這麼一個大文件後，你會想要對它進行壓縮，-z 選項是專門作這個的。因此，tar 提供了一個非常好的在互聯網上發佈程序或軟件的方法。你會發現 Linux 界裡，這個方法被廣泛的采用。

這是一個 tar 命令行的例子：

tar -zcvf myfiles.tar.gz /usr/local/bin
讓我們分析一下這行命令的具體意思：

tar
命令名。
-
告訴 tar 接下來的是選項。
z
告訴 tar 自動使用 gzip 壓縮，如果你用了這個選項，最好後綴使 .gz。
c
告訴 tar 建立一個新的檔案。
v
給出較多的信息 (verbose)，就是讓 tar 在工作時告訴你它在幹什麼。
f
這個選項是說：命令行的下一項是要建立的那個文件的名字或者要使用的設備的名字。如果在這裡使用 /dev/st0，它就會把備份寫到磁帶驅動設備/dev/st0 上。
myfiles.tar.gz
這是將要建立的文件的文件名。
/usr/local/bin
這是我們想要進行歸檔的那些文件或者目錄的名字，這裡可以放一個以上的名字， 以空格分隔。

你會經常在網上看到 tar.gz（或者縮短成 .tgz）這樣的文件，你可以使用一下命令把它解包裝：

tar -zxvf filename.tar.gz
