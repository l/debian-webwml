8 使用文本文件

文本文件在 GNU/Linux 系统上应用得十分广泛。它们包括从文档到配置文件的所有信息。幸好，使用它们不是太难。


1 查看文本文件

一个文本文件其实就是一个包含对人可读的文本的标准文件。除此之外，没有其它特殊的地方。另一种类型的文件──二进制文件，则需要计算机对它进行解释。

如果你安装了 less 文件传呼命令的话，你可以使用它来查看两种中任一类型的文件，（如果你没有安装它，最好装上，它非常有用）。输入 less /etc/profile 来查看一个示例的文本文件。注意，即使文件的含义是晦涩的，你仍然可以读出其中的字符。输入 less /bin/ls 来查看一个二进制文件。正如你所见的，ls 程序对人而言并不可读。

有时，你会发现以 .gz 结尾的文件。这些文件可能能够使用 zless 来查看；你可以如下一样运行命令：

zless /usr/doc/ae/changelog.Debian.gz

zless 对于查看文档来说十分有用，它通常是以 .gz 格式发布的。

这两种文件之间的差别完全就只是其所包含内容的差别，而不同于一些其它的操作系统（如 DOS 和 MacOS），它们对不同文件的处理方式都不同。

文本文件可以包括 shell 脚本、文档、版权通知，或者任何其它对人可读的文本。

顺便提及一下源代码与二进制可执行文件之间的差别。/bin/ls 是一个二进制可执行文件，你可以从 Debian 下载它，但是你也可以下载一个文本文件并用它来告知计算机怎样创建 /bin/ls。该文本文件即是源代码。把 /bin/ls 与 /etc/profile 作比较，就会明了如果有人想理解和修改一个软件，源代码有多么重要。自由软件提供给了你或者你的咨询者所有重要的源代码。


2 文本编辑器

文本编辑器是用来创建和修改文本文件内容的程序。大多数操作系统都有一个文本编辑器。DOS 有 edit，Windows 有 Notepad，而 MacOS 有 SimpleText。

Debian 提供了大量的文本编辑器。vi 和 Emacs 是两种典型的，它们都是功能最强大而且使用最广泛的编辑器。vi 和 Emacs 都非常复杂，需要一些操练，但是它们可以十分高效地编辑文本。Emacs 可以同时在终端和 X Window 系统下运行； vi 一般只运行在终端下，但是 vim 的变种带有 -g 选项，这可以允许其运行在 X 下。

更简单的编辑器包括 nedit、ae、jed 和 xcoral。nedit 和 xcoral 提供了易用的 X Window 系统图形接口。这儿还有一些 vi 的变种。另外，你也可以找到一个 GNU Emacs 的变种 XEmacs。

本书不包括任何特殊的编辑器的详细使用方法，可是我们将会简要地介绍一下 ae，因为它很小，运行速度又快，而且甚至可以在 Debian 急救磁盘上找到，所以为了应付紧急情况，它是值得一学的。当你需要执行更为繁重的编辑任务时，则可以试试 vim 或者 GNU Emacs。Emacs 提供了一种出色的交互式的指南；要阅读它，可使用 emacs 命令引导 Emacs，然后按下 F1 t 键。Emacs 对于关注于多用途或者编程编辑器的新用户来说，绝对是一种极好的选择。


3 使用 ae

你可以通过给定用户编辑的文件名来启动 ae，如下：

$ ae filename.txt

这将导出一个编辑器屏幕。该屏幕的顶端提供了一些快捷帮助；底端则显示出你正在编辑的文件。在这个编辑器中移动是十分简单的；只需使用箭头按键即可。你可以通过按下 C-x C-s 来保存文件，然后你可以通过按下 C-x C-c 来退出编辑器。一旦你习惯了该编辑器，你可以按下 C-x C-h 以关闭帮助。足够了！知道这些就可以让你进行基本的编辑了。为了进行编程或者更细致的编辑工作，你会考虑研究一下我们在前面提到过的其它编辑器。
