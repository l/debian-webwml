Security Information

Please send security-related bug reports to security@debian.org

Date reported: 14 May, 1998

Affected packages: gzip prior to those listed below

Brief description:
We were told by Michal Zalewski that gzexe as shipped with gzip uses an unsecure method decompressing executables on the fly opening a way of calling arbitrary programs. Newer versions for bo and hamm are fixing this. We recommend you upgrade your gzip package if you're using the gzexe method.

Vulnerable: Yes
